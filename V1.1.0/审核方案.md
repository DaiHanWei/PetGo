## 2. 违规判定标准

以下为三方服务关键词库配置方向和运营人工复审的判定参考基准。

### 2.1 通用违规类别（适用于帖子、评论、昵称、宠物名）

| 类别 | 说明 | 拦截级别 |
|---|---|---|
| 违法内容 | 毒品、武器、赌博引流词 | **强制拦截** |
| 侮辱/仇恨言论 | 种族歧视、宗教侮辱、性别攻击 | **强制拦截** |
| 色情/低俗内容 | 明确性词汇、色情暗示词 | **强制拦截** |
| 政治敏感内容 | 依据印尼 ITE 法配置关键词 | **强制拦截** |
| 垃圾广告 / 引流 | 手机号、WhatsApp、微信号、促销词 | 中等风险，运营配置可调 |
| 骚扰/恶意内容 | 针对个人的连续侮辱词 | 中等风险 |

> **注：** 三方服务关键词库需同时覆盖**印尼语（Bahasa Indonesia）**和**英语**，中文次要优先级（国际用户可能输入）。建议每季度复审一次关键词库。

### 2.2 昵称 / 宠物名字专项规则

除通用违规类别外，以下情形也纳入异步审核：

- 包含明显商业引流信息（手机号、账号链接）
- 模仿官方或知名品牌名称（仿冒风险）
- 纯数字/乱码（需人工判断是否恶意注册）

> **误判缓冲：** 宠物常见名字如「臭臭」「臭豆腐」「黑黑」等拟声词/昵称词，三方服务评分需在人工复审阶段重点判断，不宜因词汇外观直接拦截。

## 3. 各面板审核机制详述

### 3.1 帖子（V1.0.0 起生效）

**机制：** 发布时同步三方过滤

**流程：**
1. 用户点击「发布」
2. 服务端在写入 Feed 前调用三方审核 API（文字 + 图片）
3. **通过** → 帖子正常发布，用户无感知
4. **拦截** → 发布失败，返回提示「内容包含不当信息，请修改后重新发布」；内容不存储、不进人工队列

**SLA 要求：** 同步过滤响应时间 ≤ 2 秒（超时按通过处理，同时触发异步复查）

**超时处理：** 超时放行的内容异步加入人工审核队列，运营按优先级处理。

---

### 3.2 评论与回复（V1.1.0 起生效）

**机制：** 发送时同步三方过滤（与帖子相同逻辑）

**流程：**
1. 用户点击「发送」
2. 服务端调用三方关键词过滤（纯文字，无图片，速度更快）
3. **通过** → 评论立即显示在详情页
4. **拦截** → 发送失败；用户输入内容保留在输入框，提示「内容包含不当词汇，请修改后重试」；拦截记录不进人工审核队列

**特殊说明：**
- 一级评论与二级回复适用相同规则
- V1.1.0 评论为纯文字（≤200字），无图片审核需求
- 历史已发布评论若被用户举报，走 V1.2.0 FR-57 评论举报流程处理

---

### 3.3 用户昵称 / 宠物名字（V1.1.0 起生效）

**机制：** 提交后先放行，异步后台审核

**流程：**
1. 用户提交昵称（注册 FR-0E）或宠物名（创建档案 FR-11）
2. 内容**立即生效**，用户正常使用
3. 服务端异步调用三方文字审核服务，风险评分
4. **低风险（0–0.6）** → 自动通过，无需人工介入
5. **中风险（0.6–0.8）** → 进入人工审核队列，运营判定
6. **高风险（≥0.8）** → 进入人工审核队列，标记「高风险」优先处理

**运营判定违规后可执行动作：**

| 动作 | 说明 |
|---|---|
| 要求修改 | 向用户发送系统通知「你的[昵称/宠物名]包含不当内容，请在 7 天内修改，否则账号功能将受限」 |
| 临时屏蔽显示 | 将违规字段在其他用户界面替换为「***」，本人仍可见；用户修改并通过审核后自动解除 |

> **修改期限默认建议：7 天**（运营可根据严重程度调整为 3 天）。逾期未修改后功能受限范围：不可发帖、不可评论，但可正常浏览。具体文案需运营在上线前确认。

---

### 3.4 用户头像 / 宠物头像（V1.1.0 起生效）

**机制：** 提交后先放行，异步后台图像审核（与§3.3 昵称/宠物名机制一致，审核对象为图像）

**流程：**
1. 用户上传或更换头像（注册 FR-0E 或创建宠物档案 FR-11）
2. 头像**立即生效**，其他用户可正常查看
3. 服务端异步调用三方图像审核服务，输出风险评分
4. **低风险（0–0.6）** → 自动通过，无需人工介入
5. **中风险（0.6–0.8）** → 进入人工审核队列，运营判定
6. **高风险（≥0.8）** → 进入人工审核队列，标记「高风险」优先处理（对应§5.1 P1）

**运营判定违规后可执行动作：**

| 动作 | 说明 |
|---|---|
| 要求修改 | 向用户发送系统通知「你的[用户头像/宠物头像]包含不当内容，请在 7 天内修改，否则账号功能将受限」 |
| 临时替换默认头像 | 违规头像在其他用户界面替换为平台默认头像，本人仍可见原图；用户修改并通过审核后自动恢复展示 |

**举报路径仍保留：** 其他用户举报该账号/宠物档案时（FR-25，或 V1.2.0 FR-58），运营后台人工审核可直接查看并处置头像，与主动扫描互为补充。

**远期计划：** 沿用§4.2 图像识别通用阈值配置；若后续误判率高（如正常宠物照片被误判为违规内容），需三方服务补充宠物图像识别能力并建立白名单机制（参考§9.3 思路）。

## 5. 运营后台处置指引（AB-5 工单处理）

### 5.1 工单优先级与自动预处置（V1.1.0 新增自动下架机制）

| 优先级 | 触发条件 | 自动预处置 | 目标响应时间 |
|---|---|---|---|
| P0（紧急） | 法律违规内容 / 同一内容被举报 ≥ 10 次 | 系统自动下架，用户端立即不展示 | 2 小时内完成判定 |
| P1（高） | 三方高风险标注（≥0.8） | 系统自动下架，用户端立即不展示 | 24 小时内完成判定 |
| P1（高） | 同一内容被举报 3–9 次 | 不自动下架，维持现状展示 | 24 小时内完成判定 |
| P2（普通） | 三方中风险标注（0.6–0.8）/ 单次举报 | 不自动下架，维持现状展示 | 72 小时内处理 |

> **自动下架机制说明：** 触发 P0 或「三方高风险≥0.8」的 P1 内容进入「审核中」状态——用户端信息流/评论区/展示位不再显示，原作者本人可见并附「内容审核中」提示；内容不删除，仅暂停展示。运营在目标响应时间内完成判定：
> - **判定违规** → 转为永久下架，按§3对应面板的处置流程执行（涉及昵称/宠物名/头像参考§3.3、§3.4；涉及账号处置走举报处理规范）
> - **判定误报** → 自动恢复展示，无需用户操作
>
> **「举报 3–9 次」触发的 P1 不做自动下架**：举报次数可被恶意刷量（brigading）影响，不代表内容本身置信度判定，直接下架误伤正常内容的风险较高，维持现状展示、走人工复审后处置。

### 5.2 处置决定记录规范

运营处置工单时需记录：
- 判定依据（属于哪类违规）
- 处置动作
- 备注（如有特殊情况说明）

处置记录用于后续关键词库优化和三方服务阈值校准。

### 9.3 L3 宠物场景白名单（防误判，优先于 L1/L2 判定）

> 以下词汇在通用过滤库中可能被误判，但在 TailTopia 宠物社区语境下是正常词汇。**白名单优先级高于黑名单**——命中白名单的词不触发同步拦截，转为低风险异步审核或直接通过。

| 印尼语词 | 中文含义 | 误判风险来源 |
|---|---|---|
| anjing | 狗 | 同时是印尼语粗口，在独立出现时会被拦截 |
| bajingan | 流氓/坏蛋（有时用作宠物调皮的昵称） | 侮辱词 |
| gila | 疯了（口语，宠物调皮时常说「anjingku gila」） | 侮辱词 |
| nakal | 调皮（常见宠物名「Si Nakal」） | 轻度负面 |
| bodoh | 笨（部分宠物主人戏称宠物「si bodoh」） | 侮辱词 |
| gendut | 胖（超高频宠物名，因猫狗常被昵称为「si gendut」） | 体型歧视词库 |
| hitam | 黑色（黑毛宠物极常见名字） | 部分颜色词库会误判 |
| jorok | 邋遢（宠物调皮叫法） | 轻度贬义 |

> **白名单维护原则：** 白名单词汇应定期复审。若某词在用户提交内容中出现明显滥用（如被用于骚扰而非描述宠物），运营可将其从白名单移除并提升至 L2。
