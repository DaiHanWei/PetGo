## 1. §3.1 支付与收费体系

### FR-43 — 通用支付组件

**TC-43-01 — 支付页展示渠道选择**
- Given 用户触发任一收费场景（AI问诊解锁 / 兽医咨询 / 身份证高清图下载）
- When 支付确认组件弹出
- Then 展示 QRIS 渠道选项，用户可自由切换渠道，不强制记忆上次选择

**TC-43-02 — App 内嵌支付，不跳转外部浏览器**
- Given 用户选择 QRIS 支付
- When 支付流程开始
- Then 全程在 App 内嵌页面完成（二维码展示+轮询支付结果），不跳转外部浏览器或 WebView

**TC-43-03 — 三种状态组件通用**
- Given 用户处于任一收费场景的支付流程中
- When 支付进入"支付中"/"支付成功"/"支付失败"任一阶段
- Then 展示统一的状态组件；QRIS 支付中状态支持一定时长的轮询等待（非即时扣款）

### FR-43A — AI 问诊结果解锁机制

**TC-43A-01 — 结果始终立即生成，但绿/黄评级的具体内容需付费才能查看**
- Given 用户提交 AI 问诊照片+文字描述
- When AI 完成分析
- Then 无论用户本月免费额度是否用尽，AI 均立即完成分析：红色评级不因付费状态阻塞，完整安全提醒立即展示；绿色/黄色评级在额度用尽时，用户能看到"AI 已生成结果"这一事实（结果卡片已渲染出来，非加载中/失败状态），但等级图标与具体的观察建议/用药参考内容均处于模糊锁定状态、看不清具体内容——需付费解锁后才能查看（见 TC-43A-03/04）

**TC-43A-02 — 绿色评级：额度内自动解锁**
- Given 用户本月免费额度尚未使用，本次 AI 判定为绿色
- When 结果生成
- Then 等级图标与详细建议（观察建议+用药参考）同时自动解锁展示，解锁来源记为"免费额度解锁"

**TC-43A-03 — 绿色评级：额度用尽后锁定**
- Given 用户本月免费额度已用尽，本次 AI 判定为绿色
- When 结果生成
- Then 结果页顶部等级图标区域与下方详细建议区域一并模糊锁定，展示统一的"解锁查看完整结果（Rp10,000）"按钮

**TC-43A-04 — 黄色评级锁定规则与绿色一致**
- Given 用户本月免费额度已用尽，本次 AI 判定为黄色
- When 结果生成
- Then 等级图标与详细建议同样一并锁定；倒计时（条件倒计时协议）本身在解锁前不展示

**TC-43A-05 — 红色评级任何情况下免费展示（不可妥协护栏）**
- Given 用户本月免费额度已用尽，本次 AI 判定为红色
- When 结果生成
- Then 红色半屏强提醒（FR-3）立即完整免费展示，不受付费门槛影响；本次记为"超额使用"用于后台监测，但不阻拦展示

**TC-43A-06 — 症状摘要始终可见**
- Given 结果的等级图标/详细建议处于锁定状态
- When 用户查看结果页
- Then 用户自己提交的症状文字描述始终可见，不属于锁定范围

**TC-43A-07 — 解锁后永久生效，不重复收费**
- Given 用户已通过 Rp10,000 支付解锁某条 AI 问诊详情
- When 用户再次查看该条历史记录
- Then 详情直接展示为已解锁状态，无需重复支付

**TC-43A-08 — 解锁方式可选真钱或 PawCoin**
- Given 用户点击"解锁查看完整结果"按钮
- When 支付方式选择界面展示
- Then 同时展示 QRIS 真钱支付与 PawCoin 支付两个选项，互不强制

**TC-43A-09 — PawCoin 余额不足时的兜底展示**
- Given 用户选择 PawCoin 支付但余额不足
- When 支付方式选择界面展示
- Then 仅展示真钱支付选项，或引导用户前往充值

**TC-43A-10 — 解锁来源字段三选一且不可覆盖**
- Given 某条 AI 问诊记录已写入解锁来源（未解锁/免费额度解锁/付费解锁）
- When 客服后续对该记录做人工补偿等操作
- Then 补偿走独立的补偿记录，不回写/覆盖原有解锁来源字段

**TC-43A-11 — 付费解锁记录支付渠道子字段**
- Given 用户通过付费方式解锁 AI 问诊详情
- When 查看该条记录的完整数据
- Then 解锁来源记为"付费解锁"，并额外记录支付渠道子字段（QRIS 或 PawCoin），但该子字段不影响解锁来源三选一判定本身

**TC-43A-12 — PawCoin 支付后服务未交付的处理**
- Given 用户使用 PawCoin 支付解锁，但因系统故障导致生成失败
- When 故障发生
- Then 消耗的 PawCoin 原路退回 PawCoin 余额（不退真实货币），该次不在订单页创建订单记录，仅体现在 PawCoin 账本流水中

**TC-43A-13 — 生成失败不创建问诊记录**
- Given 用户提交 AI 问诊请求，但因超时/异常导致生成失败
- When 用户重新提交
- Then 失败的请求不产生任何问诊记录；重新提交视为全新请求

**TC-43A-14 — 生成成功与自动解锁为同一原子操作**
- Given 用户本月免费额度内，AI 判定为绿色/黄色
- When 结果成功生成
- Then 详情立即同步写入解锁来源"免费额度解锁"，即使用户未打开详情页或立即关闭 App，下次查看仍展示为已解锁

### FR-43B — 每月免费额度判定

**TC-43B-01 — 每月首次自动免费解锁**
- Given 用户本自然月（WIB 时区）尚未使用过免费额度
- When 用户首次需要解锁 AI 问诊详细建议
- Then 自动免费解锁，无需用户额外操作

**TC-43B-02 — 每月 1 日 00:00（WIB）重置**
- Given 当前时间跨越了自然月边界
- When 到达次月 1 日 00:00（WIB）
- Then 用户的免费额度状态重置，与用户注册日期无关

**TC-43B-03 — 额度判定对象为用户，不分宠物**
- Given 单账号下（V1.1.0 单宠物场景）
- When 用户已使用本月免费额度
- Then 该账号本月所有 AI 问诊均需付费解锁，不因宠物档案切换而重新获得免费额度

**TC-43B-04 — 免费额度不可留存/放弃**
- Given 用户本月免费额度尚未使用
- When 用户提交 AI 问诊但只看了等级、未细看详情
- Then 系统视为已使用本月免费额度，不提供"放弃本次、留到以后"的选项

**TC-43B-05 — 生成失败不消耗额度**
- Given 用户本月免费额度尚未使用
- When 提交后因网络异常导致生成失败
- Then 本月免费额度不被消耗；用户重新提交并成功出结果后才计入消耗

**TC-43B-06 — 红色评级同等消耗额度**
- Given 用户本月免费额度尚未使用，本次判定为红色
- When 成功生成结果
- Then 视为消耗了本月免费额度机会（与绿/黄一致），即使红色安全提醒本身始终免费展示

**TC-43B-07 — 并发提交的原子性**
- Given 用户在极短时间内重复点击提交，触发多次"成功生成结果"
- When 后端并发处理这些请求
- Then 同一用户的额度状态加锁读写，不允许两条结果同时被判定为"本月首次"

**TC-43B-08 — 额度消耗与解锁写入同一事务**
- Given AI 已返回完整分析
- When 落库（写入问诊记录、扣减额度）这一步本身失败（如数据库异常）
- Then 视为整体提交失败，对用户展示与生成失败一致的错误提示，不展示"已生成但未保存"的结果

### FR-43C — 红色评级安全护栏（不可妥协项）

**TC-43C-01 — 红色提醒不受任何付费改动影响**
- Given 产品/工程对 AI 问诊收费功能做出任意改动
- When 触发红色评级
- Then 红色安全提醒仍立即免费展示，不因等待支付而延误

**TC-43C-02 — 红色评级触发次数记录但不拦截**
- Given 用户在短时间窗口内被多次判定为红色评级
- When 系统记录触发次数和频率
- Then 该数据仅用于后台运营监测（不实时拦截或限制用户提交），红色提醒每次仍立即免费展示

**TC-43C-03 — 超额使用标记独立于解锁来源**
- Given 用户免费额度已用尽后又触发一次红色评级
- When 系统记录本次为"超额使用"
- Then 该标记不写入 FR-43A 的解锁来源字段（因红色本身无详细建议、无锁定问题），仅作为独立监测维度

### FR-43D — 兽医咨询收费（免费入队，接单后限时支付）

**TC-43D-01 — 提交请求不预先扣费**
- Given 用户提交兽医咨询请求
- When 请求进入待接单队列
- Then 不产生任何资金变动，广播给在线兽医

**TC-43D-02 — 无人接单超时不产生订单记录**
- Given 用户请求等待 1 分钟无兽医接单
- When 超时触发
- Then 请求直接从队列消失，不生成/保留任何订单记录，不涉及退款/手续费；用户可立即点击"重新匹配"

**TC-43D-03 — 用户接单前主动取消**
- Given 用户请求处于待接单队列中
- When 用户点击取消并二次确认
- Then 请求从队列消失，因未曾扣费，不涉及退款

**TC-43D-04 — 取消与接单的并发竞态：先到先得**
- Given 用户点击取消的同时兽医执行接单动作
- When 后端处理这两个并发请求
- Then 以服务端实际落库时间戳为准；接单生效则取消操作不再生效，进入限时支付环节

**TC-43D-05 — 兽医接单后弹出限时支付页**
- Given 兽医接单成功
- When 系统响应
- Then 立即向用户展示限时支付页，限时窗口为 1.5 分钟，扣费金额统一 Rp50,000

**TC-43D-06 — 支付方式含 PawCoin，余额不足可跳转充值**
- Given 用户在限时支付页选择 PawCoin 支付但余额不足
- When 点击"余额不足，去充值"
- Then 跳转充值流程，限时支付倒计时暂停，充值完成返回后计时恢复（非重新计满）

**TC-43D-07 — 支付成功即建立会话**
- Given 用户在限时窗口内完成支付
- When 支付成功回调
- Then 会话立即建立，用户与兽医进入咨询

**TC-43D-08 — 支付失败/超时未支付：接单作废并自动重新广播**
- Given 用户在限时窗口内支付失败或未完成支付
- When 限时窗口结束
- Then 本次接单作废，不产生扣费、不涉及退款；请求默认自动重新广播给其他在线兽医，用户无需重新提交表单

**TC-43D-09 — 重新匹配重试上限**
- Given 同一请求因超时/未支付累计触发重新广播超过 5 次或累计等待超过 30 分钟
- When 用户查看"重新匹配"按钮
- Then 按钮文案变为"多次未匹配成功，建议稍后再试"，仍允许继续重试，同时后台标记该订单供运营核查

**TC-43D-10 — 会话已建立后退款：唯一走人工工单的场景**
- Given 用户会话已建立后要求退款
- When 用户联系客服提交退款需求
- Then 不存在 App 内自助退款入口，须走客服人工工单判定；客服批准前用户看不到"选择退款方式"操作入口

**TC-43D-11 — 客服批准退款后，PawCoin 支付自动退回**
- Given 客服批准了某笔 PawCoin 支付订单的退款需求
- When 订单状态变为"退款处理中"
- Then 系统自动、即时原路退回 PawCoin 余额，无需用户选择，无需人工审核

**TC-43D-12 — 客服批准退款后，QRIS 支付可选退款方式**
- Given 客服批准了某笔 QRIS 支付订单的退款需求
- When 用户进入"选择退款方式"页面
- Then 可选择"转为 PawCoin 余额"（即时到账+溢价补偿）或"退回真钱"（BCA 免手续费 / OVO、GoPay 自付 Rp2,500 手续费，进入 1-3 个工作日人工审核周期）

**TC-43D-13 — 接单后限时未支付仅埋点，不做实时拦截**
- Given 用户多次出现"兽医接单后限时未支付"
- When 系统记录该行为
- Then V1.1.0 阶段不做实时拦截或限速，仅按用户维度统计触发次数供后续运营决策参考

### FR-43F — 支付失败/取消处理（通用规则）

**TC-43F-01 — 取消或失败回退到未提交状态**
- Given 用户在 AI 问诊解锁 / 兽医咨询接单扣费 / 身份证高清图下载 三个场景之一支付
- When 用户取消支付或支付失败
- Then 回退到未提交/未解锁状态，已填写内容或已生成的等级结果保留，不清空

**TC-43F-02 — 不触发后续业务动作**
- Given 支付取消或失败
- When 系统处理该结果
- Then 不解锁详情、不建立会话、不生成下载链接

**TC-43F-03 — 可重新触发支付，无需重走前序流程**
- Given 用户支付失败后
- When 用户重新点击触发支付
- Then 直接进入支付流程，不需要重新走前序流程（如重新提交问诊、重新填表）

## 2. §3.2 健康记录录入

### FR-45 — 健康记录录入入口

**TC-45-01 — 入口仅对状态 A 且已创建档案的用户可见**
- Given 用户为状态 B/C，或状态 A 但尚未创建宠物档案
- When 查看成长档案 Tab
- Then 不展示"健康记录"入口

**TC-45-02 — 列表按日期倒序统一排列**
- Given 用户已录入多条健康记录（含结构化录入与问诊存档）
- When 进入健康记录列表页
- Then 所有条目按日期倒序统一排列，不按类型分组

**TC-45-03 — 添加记录以弹层形式内嵌**
- Given 用户在健康记录列表页
- When 点击"添加记录"
- Then 以弹层形式呼出，非跳转独立页面

### FR-45A — 健康事件类型与字段

**TC-45A-01 — 事件类型枚举与"问诊"类型不可手动选**
- Given 用户在添加记录弹层中选择事件类型
- When 查看类型选项
- Then 可选疫苗/驱虫/月经/绝育/自定义，"问诊"类型不出现在可选列表中（仅系统生成）

## 6. §3.8 内容举报通知行为变更

### FR-51 — 举报处理结果通知

**TC-51-01 — 举报处理完成后统一通知举报人**
- Given 运营在管理后台完成了某条举报的处理（无论下架还是保留）
- When 处理操作完成
- Then 系统自动向举报人发送通知，标题"举报已处理"，正文"你的举报已处理，感谢你帮助维护社区环境"

**TC-51-02 — 不透露具体处理结果**
- Given 举报人收到"举报已处理"通知
- When 举报人查看通知内容
- Then 无法从通知文案判断内容是被下架还是被保留

**TC-51-03 — 点击通知跳转通知中心而非原内容**
- Given 举报人点击该条"举报已处理"通知
- When 系统响应跳转
- Then 跳转至通知中心列表，不跳转被举报内容详情页（内容可能已不存在）

### FR-52 — 站内信投诉发送

**TC-52-03 — 提交后可在"我的反馈"查看历史与状态**
- Given 用户已提交一条投诉
- When 用户进入"我的反馈"列表

## 8. §3.10 兽医工作台收费适配

- Given 用户在倒计时结束前完成支付
- When 支付成功

**TC-53C-03 — 被退款咨询的到手金额变更**

- When 兽医再次查看该条记录

- Then 仅可能为：待接单/待支付/进行中/已完成/退款处理中/已退款，不出现"已暂停""已取消"

- Then 订单状态回到"待接单"（不产生新订单 ID），系统自动重新广播

**TC-54A-04 — 待接单状态下支付金额展示占位文案**

**TC-54A-05 — 已退款状态展示到账细节**

- When 用户查看该订单
- Then 展示退款方式、收款渠道（如适用）与到账金额（非 BCA 渠道已扣除 Rp2,500 手续费）及到账时间
