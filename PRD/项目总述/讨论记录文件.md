### 🔐 用户登录与引导（User Auth & Onboarding）
> 对应 PRD §4.0，FR-0A 至 FR-0H。如需修改登录功能，以本节为起点。

#### 核心决策记录

| 决策项 | V1 方案 | 决策原因 | V2 规划 |
|--------|--------|---------|---------|
| 登录方式 | **仅 Google 登录** | 降低 V1 接入复杂度，Google 在印尼年轻用户中覆盖广 | 手机号+OTP（WhatsApp/SMS）、Apple 登录 |
| 登录时机 | **先浏览，触发功能时登录** | 降低冷启动拦截率，让用户先感受内容价值 | — |
| 注册/登录入口 | **合并**（Google 首次授权=自动注册）| 减少用户判断成本 | — |

#### 完整引导流程

```
App 启动 → Splash → 首页（未登录可浏览）
                          │
              ┌───────────┴────────────────────────┐
              │ 浏览到第3页时（FR-0B）               │
              ↓                                    │
         软性登录推荐浮层                            │
         主CTA显著 / 关闭按钮小且颜色浅              │
         关闭后：本次 session 不再弹                 │
              │                                    │
              └────────────────────────────────────┘
                          │
         触发核心功能时（FR-0C）
         问诊 / 发布 / 档案
                          ↓
                  强引导登录弹窗
                          ↓
                   Google 登录（FR-0D）
                          │
              ┌───────────┴──────────────┐
            老用户                      新用户
         回到触发点继续                     │
                               Step 1：昵称确认（FR-0E）
                               默认 Google 账户名，可修改
                                          ↓
                               Step 2：宠物状态选择（FR-0F，必选）
                          ┌──────────────┼──────────────┐
                          A              B               C
                      有宠物          计划养宠          爱好者
                          ↓              ↓               ↓
                    档案创建引导      直接进首页        直接进首页
                    （FR-0G）        无提示条          无提示条
                    可跳过
                    ┌──────────┐
                  完成        跳过
                    ↓           ↓
                  首页       首页+提示条（FR-0H）
```

#### 首页提示条逻辑（FR-0H）

| 项目 | 规则 |
|------|------|
| 触发对象 | 选了「有宠物」(A) 但未完成档案上传的用户 |
| 显示位置 | 首页顶部固定区域 |
| 关闭行为 | 当次 session 关闭后不再显示，下次重启继续 |
| 频次上限 | **前3次重启均显示**，第3次关闭后永不再显示 |
| 自动停止 | 完成档案上传后立即停止 |
| 不触发 | 选 B（计划养）或 C（爱好者）的用户 |

#### 宠物状态三分类说明

| 状态 | 标签 | 首页内容方向 | 档案提示条 |
|------|------|------------|---------|
| A | 我有宠物，上传宠物档案 | 健康/问诊/档案为主 | 有（跳过建档时） |
| B | 暂无宠物，但计划养 | 养宠知识/选宠指南为主 | 无 |
| C | 宠物爱好者，无养宠计划 | 宠物内容/社区为主 | 无 |

> 用户可在「个人设置 → 宠物状态」中随时修改分类。

#### 如需修改登录功能时的检查清单

- [ ] 新增登录方式（如手机号 OTP）→ 更新 FR-0D，新增对应 FR
- [ ] 调整软性弹窗触发页数 → 修改 FR-0B
- [ ] 调整提示条显示次数 → 修改 FR-0H 频次规则
- [ ] 增加宠物状态分类 → 修改 FR-0F 及首页内容方向逻辑
- [ ] 修改昵称规则 → 修改 FR-0E

### 🏥 专业问诊（Konsultasi Kilat）

| # | 功能 | 核心说明 |
|---|------|---------|
| 1 | **AI 多模态分诊** | 用户上传宠物照片/视频，AI 给出危险等级评估（绿/黄/红） |
| 2 | **黄色区间条件倒计时** | AI 不说「不严重」，只说「接下来X小时是重点观察期」，给出明确升级触发条件（如：「2小时内若再次出现→立即就医，若未出现→明早预约即可」） |
| 3 | **升级问诊入口** | AI 评估后弹出：「不放心？想确定用药方案？找权威医生还能帮你做什么」 |
| 4 | **兽医在线图文问诊** | Rp30,000/次；AI 问诊费可抵扣（已付 18K 的用户升级只需补差价 12K，非重新付 30K） |
| 5 | **新手兽医支撑系统** | 接单时可查看同类症状的历史判断数据库；无历史数据时提供 AI 参考答案；对话框显示免责提示「已告知用户本建议为参考，最终决策权在用户」 |
| 6 | **实时收入显示** | 兽医接单时对话框显示本次纯收入（如：「本单收入：Rp18,000」） |
| 7 | **V1 供给策略（运营）** | 平台 40% / 兽医 60%，吸引 IPB/UGM 网红兽医学生作为 KOL 自发传播 |

**定价结构：**
- AI 问诊：Rp18,000/次
- 专业兽医问诊：Rp30,000/次
- AI 问诊升级兽医：补差价 Rp12,000（不是重新付 30K）

**核心设计原则：**
- 平台是「工具提供方」，不是「医疗建议方」，责任在用户和兽医之间
- 用户自主决策，平台提供双向反馈渠道（举报不专业兽医 / 兽医申诉恶意差评）

### 📖 成长档案（Paspor Tumbuh Kembang）

| # | 功能 | 核心说明 |
|---|------|---------|
| 13 | **三模块统一发布入口** | 单一「+」入口，编辑页内选择：日常分享 / 专业知识科普 / 成长日历；轻量如 Instagram 发帖 |
| 14 | **成长日历——快乐时刻记录** | 照片+文字+时间，记录宠物生命中的美好瞬间；V1 聚焦「快乐时刻」 |
| 15 | **对外分享宠物名片** | 精美 H5 分享页：顶部宠物头像+名字+品种+一句话介绍，下方精选照片流；非用户只能看预览，完整内容需下载 App；天然拉新工具 |
| 16 | **活动群聊照片认领** | 群聊内他人拍的照片，宠物主一键认领，自动存入对应成长档案 |
| 17 | **问诊记录存档（用户确认）** | 问诊结束后询问：「是否将本次咨询存入 [宠物名] 的健康档案？」由用户决定，非强制 |

**目标用户：**
- 主：炫耀型——已在 Instagram 疯狂晒宠物的用户；成长档案是比 Instagram 更专属的「宠物主页」

**核心差异化：**
- Instagram = 高光橱窗（公开、社交）
- 成长档案 = 完整生命记录（私密+可分享，功能型）
- 数据沉淀 = 用户转换成本（宠物的历史记录在这里，不愿意换 App）

## 三、关键洞察与设计决策

### 专业问诊核心洞察

1. **「黄色区间」是最难设计的区间**——用户不需要「不严重可以等」，需要「接下来怎么观察、什么情况立刻行动」的可执行协议

2. **两类付费用户截然不同**
   - 焦虑型：AI 说可以等，但需要真人确认才能安心（买的是情绪）
   - 务实型：需要具体用药方案（买的是可执行建议）

3. **二次收费触发「敲诈感」**——解法：AI 问诊费抵扣，升级只补差价

4. **兽医供给经济模型已验证**：碎片时间兼职月收入 Rp3.7M > 全职助理月薪 Rp2-3M；V1 用 60% 分成 + 实习证书吸引 KOL 兽医学生

5. **平台定位清晰**：工具方 ≠ 医疗责任方；AI + 兽医签名维持专业感知层

### 宠物活动攒局核心洞察

6. **现有活动已是商业驱动**——雅加达宠物活动由场地+品牌联合举办，Instagram 发帖，WhatsApp 报名；平台角色是「聚合入口」而非「创造市场」

7. **三边市场**：宠物主（参与）× 组织者（工具需求）× 品牌/场地（曝光需求）

8. **V1 先给工具不推变现**——免费群聊+报名管理先让 Kevin 依赖平台，V2 再商业化

### 成长档案核心洞察

9. **攒局是成长档案的内容生产机器**——群聊照片认领机制让活动自动生成档案内容

10. **对外宠物名片是最强自然拉新工具**——「炫娃」动机 + 预览驱动下载，每个炫耀型用户都是分发渠道

11. **数据壁垒 > 功能壁垒**——宠物的历史记录积累越多，用户迁移成本越高；V2 的问诊历史+健康趋势会进一步锁定

## 四、三个功能的连接点（重要）

**V1 两功能连接点：**

```
专业问诊
    │ 问诊结束 → 用户确认 → 存入健康档案
    ↓
成长档案
    │ 宠物名片 → 分享 → 拉新
    └ 月费会员打通两个功能（V2）
```

**V2 加入攒局后的三功能闭环：**

```
专业问诊 → 成长档案 ← 群聊照片认领 ← 宠物活动攒局
```

V1 通过「问诊记录存档」形成数据闭环；V2 攒局加入后，「群聊照片认领」构成内容生产机器，月费会员打通三个功能实现商业闭环。
