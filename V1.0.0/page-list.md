## Step C：关键页面功能描述（含全状态处理）

以下页面承载核心用户旅程，逐一描述正常态 + 5 类非正常态的 UI 与交互处理。

---

### P-02 · 首页 Feed（访客态）

**正常态**
- Tab Bar 显示，所有 Tab 可点击；点击「成长档案/问诊/我的」触发 P-04 登录引导弹窗
- Feed 流展示社区内容卡片（日常分享/专业科普/成长时刻），图文混排
- 无顶部搜索栏（V1.0.0 不做搜索）
- 顶部 Logo + 铃铛图标（访客态铃铛点击 → P-04 登录引导）

**PL 页面加载中**（冷启动首屏）
- 展示 3 张内容卡片骨架屏（灰色矩形，高度与卡片等高，圆角与 `--radius-xl` 一致）
- 骨架屏使用 shimmer 动画（从左到右）
- Tab Bar 正常显示，不骨架化

**L 内容加载中**（上拉加载更多）
- 列表底部显示 loading spinner（`--violet-500` 色）
- 已加载内容不抖动

**E 数据为空**（Feed 内容为空，极端情况）
- 居中显示插画 + 文案：「Komunitas TailTopia baru mulai berkumpul — nantikan lebih banyak cerita di sini!」
- 副文案：「Daftar sekarang dan jadilah yang pertama berbagi!」
- 「Daftar」CTA 按钮触发 P-04

**F 加载失败**（网络断开/服务异常）
- 展示错误插画 + 「Koneksi bermasalah」
- 「Coba lagi」重试按钮，点击重新请求 Feed
- 不清空已加载内容（若有缓存则展示缓存）

---

### P-10 · 首页 Feed 主页（登录态）

**正常态**
- 顶部：Logo 左 + 铃铛（未读红点）右
- 顶部紧接：宠物档案提示条（P-11，无档案时显示）
- Feed 流：竖向滚动，卡片类型混排
- 内容卡片结构：作者头像/昵称 + 图片（最多展示1张，多图显示数量标记）+ 文字摘要（2行折叠）+ 下方互动栏（仅图标，不显示具体数字，数字在 P-12 展示）
- 无下拉刷新动画（pull-to-refresh 待 1.1.0 规划）

**PL 页面加载中**
- 3 张卡片骨架屏，shimmer 动画，`#F8F2FF` shimmer 底色 → `#FFFFFF` 高光（与 `--color-bg-subtle` 对应）

**L 内容加载中**（翻页）
- 列表末尾：48pt 高 loading 区域，`--violet-500` 圆形 spinner

**E 数据为空**
- 不应出现于登录用户（Feed 始终有内容），若出现（内容全部被删除等极端情况），展示文案引导发布：「来发布第一条内容吧！」+ CTA 跳转「+」

**F 加载失败**
- Toast：「无法加载内容，请检查网络」（Toast 样式：底部弹出，3秒自动消失）
- 内容区显示「Muat ulang」按钮

---

### P-12 · 内容详情页

**正常态**
- 顶部导航栏：左「←」返回（保持 Feed 滚动位置）；右「···」菜单（他人内容含「举报」，自己内容含「删除」）
- 作者头像 + 昵称（点击 → P-14 迷你主页卡）
- 完整图文内容（多图左右滑动，角标「2/5」）
- 互动栏：❤ 点赞数 / 💬 评论数（点击评论数跳至评论区）
- 评论区：按时间正序，一级评论 + 折叠二级回复
- 固定底部评论输入框（未登录点击 → P-04）

**PL 页面加载中**
- 整页骨架：头像圆形骨架 + 2行文字骨架 + 矩形图片骨架 + 3行评论骨架

**L 评论区加载中**
- 首次加载10条，超过10条显示「Lihat lebih banyak komentar」
- 加载中：评论区 spinner，不影响内容正文显示

**E 评论区为空**
- 文案：「Jadilah yang pertama berkomentar」
- 点击文案 → 聚焦底部评论输入框

**S 评论发布成功**
- 新评论即时插入列表底部，无全页刷新
- 底部输入框清空，键盘收起

**F 评论发送失败**
- 输入框内容保留（不清空）
- 在输入框上方显示内联错误：「Gagal mengirim, coba lagi」
- 旁边显示「↺」重试图标，点击重新发送

### P-18 · AI 分诊 - 症状上传页

**正常态**
- 顶部：「← Kembali」+ 页面标题「Konsultasi AI」
- 描述文字输入区（placeholder：「Ceritakan gejala hewan peliharaanmu...」，最多500字）
- 图片上传区（最多3张，点击触发 P-26 权限弹窗/相册）

## 非正常态设计规范（跨页面通用）

### 骨架屏（Skeleton / PL·L）

```
颜色：      底色 #F0EDF5（violet-50 加深版），shimmer 高光 #FEFEFE
动画：      从左→右线性渐变，duration 1.2s，infinite
覆盖范围：  文字 → 高度匹配行高的矩形；图片 → 原比例矩形；头像 → 圆形
注意：      骨架屏不含真实内容，不显示实际字数
```

### 空状态（Empty State / E）

```
原则：      不出现「暂无数据」字样（BMAD 明确禁止）
结构：      插画（Lottie 或静态 SVG，宠物相关主题）+ 标题文案 + 副文案 + CTA 按钮
CTA：       指向最近的解决路径（发布内容/创建档案/重新搜索等）
插画库：    统一使用品牌插画集（待设计资产产出）
语调：      积极、引导，传递「这里很快就会热闹起来」的期待感
```

### 操作成功（Success / S）

```
轻量操作：  Toast（底部弹出，3秒自动消失，`--green-500` 左侧色条）
重量操作：  全页反馈（P-31 建档庆祝/P-35 里程碑庆祝），含 Lottie 动效
即时反馈：  UI 状态直接翻转（点赞红心/里程碑徽章变色），无 Toast
```

### 操作失败/错误（Error / F）

```
原则：      错误信息具体、不使用「未知错误」
网络错误：  「Koneksi bermasalah, coba lagi」+ 重试按钮
服务错误：  「Terjadi kesalahan, silakan coba beberapa saat lagi」
内容错误：  内联展示（表单字段下方），不弹窗
用户输入：  保留输入内容，不强制清空
```

### 按钮加载态（Button Loading / BL）

```
视觉：      按钮内文字替换为圆形 spinner（`#FFFFFF`，直径与字高等高）
状态：      按钮 disabled（不可再次点击，防止重复提交）
背景色：    保持原按钮背景色（`--violet-500`），不变灰
宽度：      保持不变，防止按钮抖动
超时处理：  请求超过 10 秒 → 自动恢复按钮 + F 错误提示
```
